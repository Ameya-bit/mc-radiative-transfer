## 1. What a pulse profile is, and why verification comes first

A neutron star has a hot spot; the star spins; an observer measures brightness vs. rotation
**phase** φ — the **pulse profile**. Its strength is summarized by the **pulsed fraction**
`PF = (F_max − F_min)/(F_max + F_min)`. The observed profile is three ingredients multiplied
together:

1. **Geometry + gravitational light bending** — where the spot points relative to us as the star
   turns, and how gravity curves the rays so we partly see the far side.
2. **The beaming function `I(μ)`** — how surface brightness depends on emission angle `μ = cos α`.
   This is *our* physics (the scattering library); the textbook alternative is to assume the spot
   is **isotropic**.
3. **The observing regime** — we are monochromatic / bolometric.

The science result (v0.9.0) is the isotropic-vs-realistic *difference* in ingredient (2).
But a difference is only meaningful if ingredient (1) is provably correct: if the geometry or the
bending had a bug, a disagreement with a real star could be the bug, not the physics. So v0.8.0
ships the machinery **and** locks ingredient (1) against a closed form (the analytic check) — no new
physics, just "is the machine correct?". The beaming swap is wired in but left at its isotropic
default.

## 2. The physics, all closed-form

Everything is in the **Schwarzschild + slow-rotation** regime (no oblateness, no Doppler), which
is adequate for the low-spin verification cases and a differential result.

**Geometry — angle between spot normal and line of sight, as the star rotates:**

```
cos ψ(φ) = cos i · cos θ_s + sin i · sin θ_s · cos φ
```

with observer inclination `i`, spot colatitude `θ_s`, phase `φ ∈ [0, 2π)`. The extremes are
`cos(i − θ_s)` at φ = 0 (spot nearest) and `cos(i + θ_s)` at φ = π (farthest). → `cos_psi`.

**Light bending — Beloborodov (2002):** the emission angle `α` (from the local radial normal)
maps to the geometric angle `ψ` by a linear approximation,

```
cos α = u + (1 − u) · cos ψ ,     u ≡ R_s/R = 2GM/(R c²)   (compactness)
```

`u` is the only relativity parameter; bigger `u` = more bending = more smoothing. The **Jacobian**
`d(cos α)/d(cos ψ) = (1 − u)` is *constant* — that is what makes the approximation clean and cheap,
and what `test_bend_jacobian_is_constant_one_minus_u` pins down. → `bend`.

**Visibility:** the spot is visible while `cos α ≥ 0`, i.e.

```
cos ψ ≥ −u/(1 − u)
```

Because the threshold is **negative**, this admits `ψ > 90°`: we "see around the star". For the
figure's geometry the far-side minimum `cos ψ = cos 105° ≈ −0.26` is hidden at `u = 0` but becomes
visible once `u ≳ 0.21` lifts the threshold past it. → `visibility_threshold`.

**Observed (bolometric) flux from a point spot** (proportional, slow-rotation form; see Poutanen &
Beloborodov 2006):

```
F(φ) ∝ (1 − u) · I(μ = cos α) · cos α        when visible, else 0
```

The local emission cosine *is* `μ = cos α`, so this is exactly where the beaming library is looked
up in the isotropic-vs-realistic comparison. The **isotropic baseline** replaces `I(cos α)` with a
constant; the `cos α` projection factor stays. That isolated swap is the whole experiment. →
`point_spot_flux`, `compute_profile`, `pulsed_fraction`.

## 3. The analytic check — the load-bearing verification

For an isotropic spot `I ≡ 1`, the flux collapses to a pure geometry expression:

```
F(φ) ∝ (1 − u)(u + (1 − u) cos ψ)
```

which is monotonic in `cos ψ`, so (when the spot never sets) the extremes sit at φ = 0, π and the
pulsed fraction has a closed form:

```
PF = (1 − u)(cos ψ_max − cos ψ_min) / (2u + (1 − u)(cos ψ_max + cos ψ_min))
```

`analytic_isotropic_pf` implements this and **refuses** (raises) when the geometry eclipses the
spot, because then the minimum is an eclipse (F_min = 0) and the extremes are no longer at φ = 0, π.

The test (`tests/test_pulse.py`) compares the full numerical pipeline against this closed form,
which is re-derived **inline in the test** from `cos(i ∓ θ_s)` so the two implementations share no
code:

![Analytic check: numerical pipeline vs. closed form (residual at machine precision), and the eclipse→smoothing pulsed-fraction curve](../../data/pulse_profile_analytic.png)

- **(A)** `compute_profile` (red points) lands exactly on the closed-form flux (blue), with the
  residual flat at **max |Δ| = 0** to machine precision — the geometry + bending + projection are
  bug-free.
- **(B)** Pulsed fraction vs. compactness for one geometry (`i = 45°, θ_s = 60°`). While the spot
  is eclipsed (`u ≲ 0.21`) PF is pinned at 1 (F_min = 0). Once bending lifts it into permanent view,
  the closed form takes over (blue points on the green curve) and PF **falls** as `u` grows —
  `PF = 1.000 → 0.783 (u=0.3) → 0.600 (u=0.4)`. That decline *is* gravitational smoothing: stronger
  bending shows more of the star at once, flattening the pulse.

Two settings are tested, including one with `ψ_max > 90°` (visible around the back) where the
visibility clip and the eclipse path matter — exactly the corners where a naive implementation
breaks. The agreement is far inside the < 1 % acceptance target (it is exact to rounding).

## 4. The design seam the beaming comparison will use

The single decision that makes the isotropic-vs-realistic comparison a one-line change:
`point_spot_flux(..., beaming=None)` takes an **injected** `I(μ)` callable, defaulting to isotropic.

```python
# isotropic baseline (v0.8.0):
point_spot_flux(phase, i, theta_s, u)                    # beaming=None → I ≡ 1
# realistic beaming (v0.9.0):
point_spot_flux(phase, i, theta_s, u, beaming=I_of_mu)   # I_of_mu interpolates the τ-library
```

`test_beaming_callable_modulates_flux_but_not_geometry` already locks in that a constant beaming
reproduces the isotropic flux bit-for-bit, i.e. the geometry code is **identical** whether the
brightness law is isotropic or the scattering library. That is precisely the controlled experiment
the acceptance checklist demands ("isotropic and `I(μ; τ)` runs share identical geometry code; only
the brightness term differs"). Nothing in the comparison touches `cos_psi`, `bend`, or the
visibility clip.

## 5. Validation and status

- **The analytic check passes** to machine precision for two geometries (always-visible and
  eclipsing / see-around-the-back), against an inline-derived closed form — the geometry, bending,
  visibility, and integration are verified.
- **Unit suite: 35/35 pass** — 12 new pulse tests on the pure helpers and the analytic-check
  benchmarks, on top of the existing 23.
- **Deterministic:** no RNG, so the model and figure are identical on every run (no seeding needed,
  unlike the Monte Carlo engine).
- **Physics caveat (honest framing):** for an *isotropic* spot the flux *is* the closed form, so
  the analytic check validates the **plumbing** — phase grid, the `cos ψ` formula, the bending map,
  the visibility clip, and the PF extraction (including the eclipse case). It is not a test of new
  physics; the new physics (scattering beaming) enters in the isotropic-vs-realistic comparison,
  which is *why* that comparison needs this verified geometry underneath it.

**Next:** the code-comparison check reproduces a low-spin Bogdanov (2019, L26) case to its stated
tolerance (**v0.8.1**); then the isotropic-vs-realistic comparison swaps in `I(μ; τ)` for the
headline ΔPF result (**v0.9.0**).
