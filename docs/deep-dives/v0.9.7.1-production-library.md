## 1. What changed

The old library (`scripts/tau_sweep.py`, v0.6.0) ran a flat **200,000 injected photons at
every τ** on a single sequential RNG, and stored one point estimate per curve with no error
bar. v0.9.7 showed why that was wrong in three ways (escaped-photon currency, the noise-driven
τ = 30 dip, the missing ΔPF error bar) and prescribed a fix. This release implements it.

`tau_sweep.py` is reworked into an **escape-matched, 5-seed, process-parallel** builder:

- **Escape-matched injection.** The I(μ) histogram is built only from *escaped* photons, and
  the escape fraction falls 92% → 4% from τ = 0.1 to 30. Instead of a fixed injected count
  (which gave 22× worse statistics at τ = 30), each row targets a uniform **4×10⁵ escaped
  photons**, injecting more where the slab is thicker (v0.9.7 §3.4 table).
- **5 independent seeds per τ**, each on its own reproducible stream
  `SeedSequence(BASE_SEED=20260708, spawn_key=(τ×10, injected, seed))` — retiring the single
  sequential RNG that silently coupled the τ rows.
- **Process-parallel** over 30 independent (τ, seed) tasks, longest-cost-first, reusing the
  v0.9.7 worker-pool pattern. No engine transport change.

Two attention-point fixes from `pipeline-physics.md` were folded in while the file was open:

- **Fix 8a (normalization).** Each I(μ) curve is normalized by the *fitted* intercept `a` of
  I = a(1 + bμ), not the single noisy μ ≈ 0.975 edge bin — so the stored curve no longer
  inherits one bin's Monte Carlo noise. This needed the fit to expose `a`, so
  `src/mcrt/beaming.py` gains `fit_limb_darkening(...) → (a, b)`; the existing
  `fit_limb_darkening_slope` is now a thin wrapper over it (bit-for-bit unchanged for its
  callers).
- **Fix 8b (bin-center bias).** Counts are divided by the **mean escaped μ** in each bin
  (pooled over all runs, stored as `mu_centers`), not the geometric bin center. Grazing
  escapes crowd a bin's high edge, so the center over-states the μ a low-μ bin represents;
  the pooled mean μ is the low-variance, essentially τ-independent estimator of it. (Observed:
  `mu_centers[0] = 0.032` vs the 0.025 bin center — the expected upward shift.)

## 2. The run

| Injected | 79.3M photons total |
| Workers | 16 (macOS spawn pool) |
| Wall time | **71.9 min** |
| Long pole | the 5 τ = 30 tasks, ~70 min each, run in parallel on dedicated cores |

The `~30–90 min` v0.9.7 estimate held; the thick-slab tasks landed at the upper end because
per-photon scattering cost at τ = 30 is steeper than the scheduling heuristic assumed (the
heuristic is used only to order tasks, never for physics). Each row hit its escaped target
within ~1% (392k–404k escaped), confirming escape-matching delivered **uniform statistics**
across τ for the first time.

## 3. Results — b(τ) ± σ, and the gates

| τ | injected/seed | escaped (mean) | f_esc | **b ± σ (this run)** | v0.9.7 converged | Δ |

| 0.1 | 4.4×10⁵ | 402,863 | 0.916 | 0.292 ± 0.010 | 0.282 ± 0.003 | +0.010 |

| 3.0 | 1.3×10⁶ | 392,174 | 0.302 | 1.795 ± 0.032 | 1.722 ± 0.038 | +0.073 |
| 10 | 3.4×10⁶ | 396,634 | 0.117 | 1.790 ± 0.035 | 1.774 ± 0.053 | +0.016 |
| 30 | 9.5×10⁶ | 403,498 | 0.042 | 1.809 ± 0.040 | 1.756 ± 0.079 | +0.053 |

**Sanity gates (all green):**

- **b(τ) rises monotonically within error to a ~1.8 plateau.** τ = 3/10/30 = 1.795/1.790/1.809
  are statistically flat — the v0.6.0 library's τ = 30 *dip* (which had wrongly justified
  `SHAPE_TAU = 10` as "where b peaks") is gone, exactly as v0.9.7 predicted.
- **τ = 10 and τ = 30 are statistically identical** — the headline point of the redo.
- **Every row met the 4×10⁵ escaped target**, so per-row quality is uniform.
- **Error bars now ship.** Per-seed σ(I) at τ = 10 ≈ 0.025 per bin; pooled over 5 seeds
  ≈ 0.011 — inside the σ(b) ≤ 0.02 / tail ≤ 2% pooled targets.

**Honest caveat — the systematic +offset.** The new b values sit ~+0.01 to +0.07 above the
v0.9.7 reference. That reference was computed with the *old* extraction (bin-center divisor +
single-bin normalization); the production library uses **8a + 8b**, so a small, consistent
positive offset is expected *methodology*, not noise or a convergence failure. All six rows
remain within 2σ of the reference. The 8a/8b extraction is the more-correct one and is now the
single source of truth going forward.

## 4. What is stored

`data/beaming_library.npz` — 18 keys, a **superset** of the old 4-key schema so every existing
downstream loader (`anchor_lib.py`, `beaming_pulse_sweep.py`, and the anchors/phase-diagram/
finite-cap scripts through them) reads it unchanged:

- **Backward-compatible:** `tau_values`, `mu_centers` (mean-escaped-μ grid), `intensity_by_tau`
  (6×20, pooled), `b_of_tau`.
- **New — errors & per-seed:** `intensity_std_by_tau`, `b_std_of_tau`, `escape_frac_by_tau`,
  `intensity_by_tau_seed` (6×5×20), `b_by_tau_seed` (6×5), `escaped_by_tau_seed`,
  `injected_per_seed`. (Track E2's tail-sensitivity resampling needs the per-seed curves.)
- **Provenance:** `n_seeds`, `n_bins`, `mu_floor`, `base_seed`, `target_escaped`,
  `generated_utc`, and a `provenance` string.

Figures regenerated with production data + error bars: `data/beaming_tau_curves.png`,
`data/beaming_slope_vs_tau.png`.
