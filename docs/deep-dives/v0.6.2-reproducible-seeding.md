## 1. The problem: global randomness has no error bar

A Monte Carlo answer is only as trustworthy as the error bar around it, and the honest way to
get that error bar — when there is no closed-form truth for the noisy low-μ tail bins — is to
**repeat the run under independent randomness and measure the spread**. That requires two things
the old engine could not give:

- **Reproducibility.** Re-running the exact same experiment should give the exact same numbers,
  so a result can be regenerated and a figure re-made without re-deriving it.
- **Independent, controllable streams.** Each repeat must use randomness that is statistically
  independent of the others, with no hidden correlation.

The engine drew every random number from NumPy's **global** `np.random` module. Global state is
shared and order-dependent: two runs can never be made identical on demand, and "just call
`np.random.seed()` again" does not give you several *independent* streams — it gives you the same
stream restarted. There was no clean hook to seed one run without disturbing the next.

---

## 2. The change: thread an explicit `Generator`, default to the old behaviour

NumPy's modern API replaces the global module with explicit `Generator` objects you create and
pass around. The engine now accepts one:

```python
# src/mcrt/monte_carlo.py
class Simulation:
    def __init__(self, tau_total, num_photons=1000, rng=None):
        ...
        # An explicit Generator makes a run reproducible; convergence studies
        # pass per-run seed offsets. Default to a fresh Generator so each
        # Simulation is independent without touching global np.random state.
        self.rng = rng if rng is not None else np.random.default_rng()
```

That generator is threaded through injection, step sampling, and scattering, and on down into
every sampler in `utils.py`. Each sampler grew the same optional parameter with a graceful
fallback:

```python
# src/mcrt/utils.py — before
def sample_step_size():
    return -np.log(np.random.random())

# after
def sample_step_size(rng=None):
    if rng is None:
        rng = np.random          # legacy global behaviour, unchanged
    return -np.log(rng.random())
```

The fallback is the important detail: when `rng is None` the sampler uses the global `np.random`
exactly as before, so **every existing call site and unit test keeps its original behaviour
byte-for-byte**. The reproducibility is opt-in; nothing downstream had to move. The same pattern
applies to `sample_thomson_angle`, `get_random_direction`, and `rotate_vector`.

With an explicit generator in hand, the same seed reproduces a run exactly:

```python
a = Simulation(tau_total=5.0, num_photons=500, rng=np.random.default_rng(12345)); a.run()
b = Simulation(tau_total=5.0, num_photons=500, rng=np.random.default_rng(12345)); b.run()
assert a.results['escaped_mu'] == b.results['escaped_mu']   # identical, every photon
```

## 3. Independent streams: `SeedSequence.spawn`

Reproducibility is half the story; a convergence study also needs **many** runs whose randomness
is mutually independent. Restarting one seeded stream does not give that. `SeedSequence` does:

```python
children = np.random.SeedSequence(base_seed).spawn(k)        # k independent seeds
rngs = [np.random.default_rng(c) for c in children]          # k independent generators
```

`spawn` derives `k` child seed sequences whose generators are guaranteed independent (no
accidental overlap or correlation), while the whole set is reproducible from the single
`base_seed`. This is exactly what the v0.7.0 study consumes: one independent stream per
`(photon-count, seed)` pair, so the spread it measures across seeds is real statistical noise and
not an artifact of shared state.

A caveat worth recording: because the streams are assigned by a flat index, inserting a new
photon-count anywhere but the end of the grid reshuffles which child seed each later run gets.
Append new counts (or bump the base seed) when changing a grid you want to keep reproducible.

## 4. Validation and status

- **Reproducibility test:** two simulations with the same seed produce identical escape angles
  and identical absorbed counts (`test_simulation_reproducible_with_same_seed`).
- **Independence test:** two simulations with different seeds produce different realizations
  (`test_simulation_differs_with_different_seed`).
- **No regression:** because `rng=None` preserves the global-`np.random` path, the primitive and
  H-function suites are untouched. **11/11 tests pass** (the 2 new seeding tests on top of the 9
  primitive/theory tests).
- **First adopter:** `scripts/tau_sweep.py` now builds the `I(μ; τ)` library on
  `Simulation(rng=default_rng(SEED))` instead of seeding the global module, so the library is
  reproducible through the same hook the study uses.

This is infrastructure, not physics: no beaming-function number changes. It is split out as its
own version because it is the reusable foundation every future multi-seed experiment (convergence,
and later pulse-profile error budgets) stands on.

**Next:** spend this reproducibility on the real question — the photon-count convergence study
(error vs N on log-log axes, recommended N per observable), **v0.7.0**.
